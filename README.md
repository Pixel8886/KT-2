# XML Schema — КТ

## Содержимое проекта

- **notebook.xsd** — XML Schema, описывающая структуру документа `notebook`.
- **valid.xml** — корректный XML-файл, проходящий валидацию схемой.
- **no-valid-1.xml … no-valid-5.xml** — набор ошибочных XML-файлов, которые **не должны проходить** валидацию.

##  Основные требования, реализованные в схеме

- Корневой элемент — **notebook**
- `notebook` содержит:
  - один элемент **offices**
  - минимум один элемент **person**
- `office`:
  - содержит текст
  - имеет атрибут **code** (тип `ID`, уникальный)
- `person`:
  - содержит элементы `name`, `address`, `phones`
  - имеет атрибуты:
    - **id** — уникальный (`ID`)
    - **office** — ссылка на существующий `office/@code` (`IDREF`)
- `name` содержит `first` и `surname`
- `address` содержит `city`, `street`, `index` (порядок произвольный)
  - `index` — строго 6 цифр
- `phones` содержит любое число элементов `phone`
- `phone` содержит строку формата **XXX-XX-XX**
  - атрибут `type` допускает только значения: **work**, **home**

##  Проверка валидации

- **valid.xml** корректно проходит валидацию схемой.
- Каждый из файлов **no-valid-1.xml - no-valid-5.xml** содержит ошибку и **не должен проходить** валидацию:
  - повторный ID
  - несоответствие IDREF
  - неверный тип phone
  - неправильный формат номера телефона
  - некорректный индекс (не 6 цифр)
